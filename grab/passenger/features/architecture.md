# 🧩 Super-App Architecture: Grab Passenger

### 🔍 Binary Evidence
*   **Orchestration:** Modular loading of **"Grablets"** (mini-apps).
*   **Dynamic Updates:** `feature_patch` logic for real-time UI updates.
*   **Backend Hub:** `https://api.grab.com` and `https://data-api.grab.com`.

### ⚙️ Technical Logic Summary
Grab employs a highly modular "Grablet" architecture, enabling the platform to function as a Super App. Dynamic patching allows for UI and logic updates without requiring a full store release, facilitating rapid feature deployment across diverse regional markets.
